# D8.1 — Avatar Color Palette Extension

Status: Locked 2026-05-16
Amends: D8 (design system — "avatars: blue palette only" rule)
Triggered by: D7.3 (gender-derived avatar color)

## Decision

Avatar color encodes gender (amended 2026-06-14 — see D8.1a):
- Woman → pink family
- Non-binary → green family
- Man, Prefer not to say, NULL, anything else → blue family (default)

Coral remains forbidden for avatars (coral is reserved for actions/CTAs
only — that D8 rule is unchanged).

## Blue family (default — D8.2 values)

Man / prefer-not-to-say / NULL / anything else → blue, four shades (bg):
- #8DBCF1 / #5E80A3 / #14304D / #9DB8D2

Live in `src/lib/avatar.ts`. Supersedes the original D8.1 blue
`#1A3650 / #3A7CB8 / #5A9FD4` — reconciled to the D8.2 palette.

## Pink palette (finalized 2026-06-14)

Woman → pink family, four shades (bg) — each mirrors the matching blue
shade's saturation + lightness at pink hue (not muted-blue-vs-loud-pink):
- #F18EB7 / #A25D7A / #4D142C / #D29DB3

## Green palette

Non-binary only, one shade (bg):
- #246B42

Prefer not to say / NULL → blue (NOT green). See D8.1a for the rationale
(privacy + NULL/PNTS ambiguity both resolve to blue).

## Home fill dots are neutral (D7.3)

Gender-derived color is member-context — it appears in the lobby roster
only. The pre-join Home feed renders fill dots in a single neutral
(#AEBED0) so the gender mix is never leaked to non-members.

## Does NOT change

Coral action-only rule. Families are blue / pink / green only — no others.

- ✅ Avatar rendering: gender → color mapping live in `src/lib/avatar.ts`
