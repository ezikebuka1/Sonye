# D7.2 — Onboarding Form Bifurcation

Status: Locked 2026-05-16
Amends: D7 (builds on D7.1)
**Amended by: D7.4 (PENDING LOCK — drafted 2026-06-10)** — collapses the
slot-vs-generic field split; drops `general_availability` / `preferred_venues`
from the in-app form. Do not treat D7.4 as locked until explicitly stated.
Triggered by: Real-user feedback 2026-05-16 — a user who taps a shared
slot link has already revealed availability + venue preference by the act
of joining; re-asking in a form violates the Partiful Model.

## Decision

The onboarding form bifurcates by entry context:

- Path A (waitlist claim): no form. Fields pre-populated from CSV.
- Path B-slot (net-new, joining via a specific shared slot link):
  minimal form — first_name, last_name, skill_level, optional gender.
  availability and preferred_venues NOT asked; recorded NULL.
- Path B-generic (net-new, no slot context): full D7.1 six-field form
  plus optional gender.
- Path C (returning user): no form. Auth only.

Distinguishing input: presence of a slot ID in the entry route.
Slot-context → minimal form. No slot-context → full form.

## Why

The join action is stronger data than a self-reported form. The skipped
fields (availability, preferred_venues) are not used anywhere in v1 —
same test D7.1 applied to willing_to_drive. Path B-slot is the dominant
v1 entry path per the Partiful Model; optimizing it for minimal friction
is the highest-leverage conversion move.

## Name field

Two inputs: "First name" / "Last name". Stored as distinct first_name +
last_name columns (locked in D3 Step 1.2 for curator disambiguation).
Single-input-with-split was explicitly rejected — fails on mononyms and
complex names.

## Schema implications (feed D3)

- users.general_availability becomes nullable
- users.preferred_venues becomes nullable
- No new columns; no type changes beyond nullability
- v1 uses neither field for any read or write path

## Does NOT change

D7 core mechanics (slots, group 6, curation, waitlist), D7.1
(willing_to_drive cut), phone via OTP not form, sport implicit
pickleball, D3 Step 1.2 identity-resolution flow.
